
# プロダクトバックログ

---

## 1. 優先度 ★★★（最優先）

### ● 抽選リマインダー機能（v2.0）

* 毎年 / 毎月 / 毎週の3種類で期間設定
* 期間中はトップ画面でメッセージ常時表示
* 固定文言・複数メッセージ対応
* リマインダー設定の追加／削除

### ● 参加履歴管理（v2.1）

* 各イベントの参加表明履歴を保存
* 1名につきコメント1つ（最新上書き）
* 最新順（降順）で表示
* 削除不可
* コメントは改行対応

---

## 2. 優先度 ★★☆（高）

### ● カレンダースワイプ操作（v2.2）

* スマホで左右スワイプ → 月切替
* PCは従来通りボタン操作

### ● カレンダーからの自動遷移（v2.2）

* 日付選択 → 自動で新規登録画面へ
* イベント選択 → 自動で参加表明画面へ

### ● Google カレンダー API 連携（v2.3）

* Google Calendar API を使用し、イベントを直接ユーザのカレンダーへ登録
* タイトル・日時・施設名・備考を連動
* OAuth 認証によるユーザ許可後、自動登録
* 二重登録防止の簡易チェック（eventIdなど）

### ● イベント施設の場所リンク表示（v2.3）

* イベント詳細画面に Google Maps へのリンクを表示
* 施設住所を Google Maps 形式で自動生成

### ● 参加者未選択エラー表示（v2.3）

* 参加者を誰も選択しないまま登録しようとすると警告
* 入力補助のガイド文も表示

---

## 3. 優先度 ★★☆（高）

### ● 開発／本番の二重環境構築（v3.0）

※ ユーザは **常に本番環境のみ** を使用する。開発環境は開発者専用であり利用者には見せない。

* 開発者専用の **開発環境用データファイル（例：dev_reservations.csv）** を分離
* 開発環境で十分に検証後、本番へデプロイ
* **一般ユーザは環境切替を一切意識しない**
* 環境切替は開発者のみが使用する内部設定
  * 例：config.dev.yaml / config.prod.yaml を切り替えて実行
  * または、開発者が実行時に環境引数を渡して起動（ユーザには非公開）
* デプロイ後、ユーザは常に本番環境のみ利用
* 開発環境ではログ詳細モード、本番環境では保護と安定性重視
